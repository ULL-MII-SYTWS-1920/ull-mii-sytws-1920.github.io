Fault tolerant Promise.all

We’d like to fetch multiple URLs in parallel.

Here’s the code to do that:

 let urls = [
  'https://api.github.com/users/iliakan',
  'https://api.github.com/users/remy',
  'https://api.github.com/users/jeresig'
];

Promise.all(urls.map(url => fetch(url)))
  // for each response show its status
  .then(responses => { // (*)
    for(let response of responses) {
      alert(`${response.url}: ${response.status}`);
    }
  });

The problem is that if any of requests fails, then Promise.all rejects with the error, and we lose results of all the other requests.

That’s not good.

Modify the code so that the array responses in the line (*) would include

  • the response objects for successful fetches and
  • error objects for failed ones.

For instance, if one of URLs is bad, then it should be like:

let urls = [
  'https://api.github.com/users/iliakan',
  'https://api.github.com/users/remy',
  'http://no-such-url'
];

Promise.all(...) // your code to fetch URLs...
  // ...and pass fetch errors as members of the resulting array...
  .then(responses => {
    // 3 urls => 3 array members
    alert(responses[0].status); // 200
    alert(responses[1].status); // 200
    alert(responses[2]); // TypeError: failed to fetch (text may vary)
  });

P.S. In this task you don’t have to load the full response using response.text() or response.json(). Just handle fetch errors the right way.

click 3 buttons: order of events

There are 3 buttons that have click event listeners attached to them. Clicking the buttons will do the following:

  • Button 1 - put a 5 second long synchronous task on the event queue

  • Button 2 - put a 15 second long synchronous task on the event queue

  • Button 3 - put a 1 second long synchronous task on the event queue

The buttons are clicked in the following order: Button 1, Button 2, Button 3

What will happen?

  • Button 1's task executes first because it was clicked first, followed by Button 2's and then button 3's
  • All three tasks start at the same time and run in parallel
  • Button 3's task executes first because it is the quickest, followed by Button 1's' and then Button 2's task
  • Only button 1's task is executed. The other's don't execute because they were clicked while button 1 was processing

event queue and call stack

There are currently several functions on the call stack that are in the process of running.

The event queue is empty.

A mouseclick event occurs and puts a new task onto the event queue.

What happens next?

  • The mouseclick task occurs first because it is asynchronous and then the functions on the call stack resume running
  • The call stack finishes running and then the mouse click task gets put on the call stack to run
  • Both the mouseclick task and the call stack functions run at the same time
  • The mouseclick never occurs because the event happened while the call stack was not empty
